# UX Writing (Mobile)

- Keep copy concise and actionable.
- Labels should be concrete and scannable.
- Errors should explain what happened and what to do next.
- Empty states should teach users how to proceed.
- Button text should reflect outcomes ("Save changes", "Send request").

Accessibility checks:
- Do not rely on color alone to communicate meaning.
- Provide semantic labels for assistive technologies.
- Ensure dynamic announcements for important status changes when needed.
